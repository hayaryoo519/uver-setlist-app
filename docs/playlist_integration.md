# 外部サービス連携：プレイリスト作成機能 仕様書

## 1. 概要
UVERworldのライブセットリストを、ユーザー自身のSpotifyまたはYouTube Musicのアカウントに「プレイリスト」として自動作成する機能。

## 2. 対象プラットフォーム
- **Spotify**: Spotify Web APIを使用。
- **YouTube Music**: Google OAuth 2.0 & YouTube Data API v3を使用。

## 3. 基本仕様
- **作成モード**: 「非公開プレイリスト」として作成される（ユーザーのライブラリに保存）。
- **ログイン必須**: 現在の仕様では、アプリの「Crewアカウント（ログイン状態）」に連携情報を紐付けているため、ログインが必須。
- **重複防止**: 同一ユーザーによる短時間（10秒以内）の連続作成を防止。
- **履歴管理**: 過去にそのライブで作成したプレイリストのリンクを最大5件まで保持。

## 4. ユーザーフロー
1. **連携開始**: ライブ詳細ページの「Streaming Integration」セクションでプラットフォームを選択。
2. **外部認証**: 「連携して作成」ボタンを押し、Spotify/Googleの認証画面（ポップアップ/別タブ）で承認。
3. **作成実行**: 連携完了後、「プレイリストを作成」ボタンをクリック。
4. **自動生成**:
    - DB内の `spotify_track_id` / `youtube_video_id` を優先して参照。
    - IDがない場合は、楽曲タイトルで自動検索し、ヒットしたものを追加。
    - 検索結果を次回の高速化のためにDBへ書き戻し（オートマッピング）。
5. **完了**: 作成されたプレイリストへの直リンクを表示。

## 5. UI/UX デザイン
- **レスポンシブ**: PC/スマホ両対応。モバイルではボタンサイズを調整し、タップしやすいレイアウトを採用。
- **ステータス表示**: 「未連携」「作成中（ローディング）」「成功」「エラー」の状態を視覚的にフィードバック。
- **ブランドカラー**: Spotify（Emerald/Green）、YouTube Music（Red）のカラーシステムに準拠。

## 6. 技術構成
### バックエンド
- **Endpoint**: `/api/spotify/*`, `/api/youtube/*`
- **Security**: アクセストークン・リフレッシュトークンはAES-256で暗号化してDB保存。
- **Auto Mapping**: 楽曲が見つからなかった場合、次回以降の精度のために検索結果を `songs` テーブルに正規化して保存。

### データベース（主なテーブル）
- `user_spotify_tokens` / `user_youtube_tokens`: 連携情報。
- `playlist_history`: 作成したプレイリストのメタデータ。
- `songs`: `spotify_track_id`, `youtube_video_id` カラムでマッピングを保持。

## 7. 今後の拡張予定（検討事項）
- **ゲスト対応**: ログインなしでもその場限りの認証で作成できるようにする。
- **セットリスト予想のプレイリスト化**: 自分が予想したセトリをプレイリストにする機能。
